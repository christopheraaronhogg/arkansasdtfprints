Reasoned for a second
Native HTML date inputs can be quite stubborn about how they render placeholders and icons across different browsers. To really “tame” that extra gap, you often need a combination of vendor-specific selectors to strip out default padding/margins inside the <input type="date">. Below is a more aggressive override that zeros out the internal margins on each “datetime-edit” field, plus the calendar icon. This typically fixes stubborn extra spacing:

css
Copy
/* For <input type="date"> in Chrome, Edge, Safari (WebKit/Blink) */
input[type="date"] {
  -webkit-appearance: none; /* Remove default Chrome/Safari styling */
  -moz-appearance: none;
  appearance: none;
  padding: 0.5rem 0.75rem;  /* Adjust as needed for your design */
  line-height: 1.4;
  border: 1px solid var(--border-color);
  border-radius: var(--radius-sm);
  font-size: 0.9rem;
  color: var(--text-primary);
  width: 100%;
  box-sizing: border-box;
}

/* Remove default icon spacing/margins */
input[type="date"]::-webkit-calendar-picker-indicator {
  margin: 0; 
  padding: 0; 
  cursor: pointer;
}

/* Zero out internal field spacing in WebKit-based browsers */
input[type="date"]::-webkit-datetime-edit {
  margin: 0;
  padding: 0;
}

input[type="date"]::-webkit-datetime-edit-fields-wrapper {
  padding: 0;
}

input[type="date"]::-webkit-datetime-edit-text,
input[type="date"]::-webkit-datetime-edit-month-field,
input[type="date"]::-webkit-datetime-edit-day-field,
input[type="date"]::-webkit-datetime-edit-year-field {
  padding: 0;
  margin: 0;
}

/* Hide the clear (X) button in Edge/IE */
input[type="date"]::-ms-clear {
  display: none;
}
/* Remove the spin buttons in Edge/IE */
input[type="date"]::-ms-expand {
  display: none;
}
Why So Many Selectors?
::-webkit-datetime-edit-* rules target the internal building blocks of Chrome/Safari’s date field (month, day, year segments). Setting margin: 0; padding: 0; on them removes that extra offset.
::-webkit-calendar-picker-indicator addresses the calendar icon’s offset.
-webkit-appearance: none; and equivalents remove the default “native” look.
Final Tips
If you still see quirks, remember that Firefox handles date inputs differently; sometimes it falls back to a text field if your locale or date format aren’t supported.
If consistent styling is critical across browsers, many devs opt for a text input + a datepicker library (e.g., flatpickr, Bootstrap Datepicker) instead.
But with the above “all‐out override,” you should see that placeholder (mm/dd/yyyy) align without the annoying extra gap in most modern browsers. Feel free to adjust the padding and font-size for your design.
